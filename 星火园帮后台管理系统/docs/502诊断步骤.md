# 502 诊断步骤（在服务器上依次执行）

## 第一步：确认 Node 监听端口

```bash
ss -tlnp | grep -E "1111|1112"
```

- 若显示 `1112` → 端口正常
- 若显示 `1111` → 需把 Nginx 改为代理到 1111，或把 ecosystem 的 PORT 改为 1112 并重启
- 若都不显示 → Node 未启动或端口不对

## 第二步：直接请求 Node（绕过 Nginx）

```bash
curl -v -X POST http://127.0.0.1:1112/admin/api/v1/auth/signin \
  -H "Content-Type: application/json" \
  -d '{"email":"1850336901@qq.com","password":"hyq/\\911"}'
```

- 若返回 JSON（登录成功或 401 等）→ Node 正常，问题在 Nginx
- 若 `Connection refused` → Node 未监听 1112，检查端口
- 若无响应/超时 → Node 可能卡死或崩溃

## 第三步：看 Nginx 错误日志（登录时执行）

一个终端执行：

```bash
tail -f /var/log/nginx/error.log
```

另一个终端或浏览器发起登录，观察是否出现：

- `connect() failed (111: Connection refused)` → 后端端口错误或未启动
- `upstream timed out` → 后端响应超时
- 无新日志 → 请求可能未到达 Nginx 或走了其他 server

## 第四步：确认 PM2 使用的脚本和端口

```bash
pm2 show xinghuoyuanbang
```

检查：

- `script path`：是否为 `server.js` 或实际存在的入口
- `env`：`PORT` 是否为 1112
- `exec cwd`：项目目录是否正确

## 第五步：确认 Nginx 实际配置

```bash
nginx -T 2>/dev/null | grep -A5 "location /admin/api"
```

确认 `proxy_pass` 的端口与 Node 监听端口一致。

## 第六步：部署更新并重启

1. 上传最新代码（含 requestLogger 中的 `[REQUEST]` 日志、ecosystem 使用 server.js、PORT: 1112）
2. `pm2 restart xinghuoyuanbang`
3. 再次登录，同时执行 `pm2 logs xinghuoyuanbang --lines 50`
4. 若看到 `[REQUEST] POST /admin/api/v1/auth/signin` → 请求已到 Node
5. 若仍无该日志 → 请求未到 Node，重点检查 Nginx 和端口
