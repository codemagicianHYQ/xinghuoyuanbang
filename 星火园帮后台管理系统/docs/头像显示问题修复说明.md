# 用户头像手机端不显示 - 修复说明

## 问题现象

- 头像在微信开发者工具（电脑端）可以显示
- 头像在真机（手机端）无法显示

## 根因

1. **URL 中含中文路径**：如 `用户头像` 未做编码，部分手机端 WebView 解析异常
2. **URL 中含空格**：文件名如 `1771432247602_li958d gqfw.jpg` 中的空格需编码为 `%20`

## 已做修改

### 1. 后端 URL 编码（`config/domain.js`）

- 新增 `encodeFileUrl(url)`：对已有文件 URL 进行编码
- 修改 `generateFileUrl()`：生成时对路径进行编码（`encodeURIComponent`）

### 2. User 模型（`models/user.model.js`）

- 对 `avatarUrl` 增加 getter，读取时自动编码后再返回

### 3. 上传接口（`routes/upload.routes.js`）

- 本地存储回退时的头像 URL 使用 `encodeFileUrl` 编码后再返回

## 微信小程序配置（必查）

若修改后手机端仍不显示，请在**微信公众平台**检查：

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入 **开发** → **开发管理** → **开发设置** → **服务器域名**
3. 在 **downloadFile 合法域名** 中增加：
   ```
   https://files.xinghuoyuanbang.top
   ```
4. 若本地/开发环境使用 `xinghuoyuanbang.top`，也需在 downloadFile 域名中加入该域名

未配置 downloadFile 域名时，小程序无法加载外部图片，手机端会失败，而开发者工具可能因 `urlCheck` 关闭而能正常显示。

## 验证方式

1. 重新部署后端
2. 真机预览 / 体验版中查看「我的」页头像
3. 确认 `downloadFile` 合法域名已配置
