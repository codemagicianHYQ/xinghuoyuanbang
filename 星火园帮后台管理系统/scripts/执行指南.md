## 🔍 执行后验证

### 在 MySQL Workbench 中执行以下查询：

```sql
-- 1. 检查备份表是否已清理
SELECT COUNT(*) AS 备份表数量
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper'
  AND TABLE_NAME LIKE '%backup%';
-- 预期结果：0

-- 2. 检查主表是否已删除
SELECT COUNT(*) AS 主表数量
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper'
  AND TABLE_NAME IN ('campus_resources', 'market_products', 'tasks');
-- 预期结果：0（如果主表已删除）或 1（如果 tasks 主表还在）

-- 3. 统计分表数量
SELECT
    '任务分表' AS 类型,
    COUNT(*) AS 数量
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper'
  AND TABLE_NAME LIKE 'tasks_community_%'

UNION ALL

SELECT
    '校园互动分表' AS 类型,
    COUNT(*) AS 数量
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper'
  AND TABLE_NAME LIKE 'campus_resources_community_%'

UNION ALL

SELECT
    '二手市集分表' AS 类型,
    COUNT(*) AS 数量
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper'
  AND TABLE_NAME LIKE 'market_products_community_%';
-- 预期结果：每种类型 5 个分表

-- 4. 查看数据库总大小
SELECT
    ROUND(SUM((DATA_LENGTH + INDEX_LENGTH) / 1024 / 1024), 2) AS '数据库总大小(MB)'
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'campushelper';
```

## ⚠️ 注意事项

### 执行前

1. **备份数据库**（强烈推荐）：

   ```bash
   mysqldump -u root -p campushelper > campushelper_backup_$(date +%Y%m%d_%H%M%S).sql
   ```

2. **停止应用服务器**（可选，更安全）：
   ```bash
   # 在服务器目录执行
   pm2 stop all
   ```

### 执行中

1. 仔细查看每一步的执行结果
2. 如果遇到错误，立即停止并记录错误信息
3. 不要跳过任何警告信息

### 执行后

1. 验证数据库结构是否正确
2. 重启应用服务器：
   ```bash
   pm2 restart all
   ```
3. 测试应用功能是否正常

## 🚨 常见问题

### Q1: 删除备份表会影响应用运行吗？

**A**: 不会。备份表只是数据副本，不被应用使用。

### Q2: 如果删除后发现问题怎么办？

**A**: 如果提前备份了数据库，可以从备份恢复。

### Q3: 可以只删除部分备份表吗？

**A**: 可以。编辑 `safe_clean_backups.sql`，注释掉不想删除的 `DROP TABLE` 语句。

### Q4: 主表引用检查发现了引用怎么办？

**A**: 记录引用信息，手动执行以下命令删除外键：

```sql
ALTER TABLE 表名 DROP FOREIGN KEY 外键约束名;
```
